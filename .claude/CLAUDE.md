# CLAUDE.md — Scholar Research Workspace

## Core constraint: verify before deciding

Assume all known knowledge may be wrong, outdated, or irrelevant to the current context. Before making any meaningful decision, recommendation, or claim:

1. **Search first** — use Exa MCP (`web_search_exa`), Semantic Scholar, arXiv, or local files to find current, relevant information.
2. **Cross-check** — a single source is not enough. Look for corroborating or conflicting evidence before settling on an answer.
3. **Cite sources** — when presenting information as fact, include where it came from and when (year/date).
4. **Admit uncertainty** — if no authoritative source can be found, state the uncertainty clearly rather than relying on internal knowledge alone.

This applies especially to:
- Claims about what a paper, model, or method does
- API behavior, library defaults, or version-specific features
- Benchmarks, leaderboard positions, or SOTA comparisons
- "Common knowledge" that may have been superseded

Internal knowledge is a starting point for search queries, not a source of truth.
